# AI PROMPTER 사용법

음성을 인식해 대본을 자동으로 따라 넘겨주는 AI 텔레프롬프터입니다.
이 문서는 실제 사용 흐름과 세부 설정, 그리고 음성 인식을 잘 쓰는 요령을 정리합니다.

---

## 1. 실행하기

- **앱(.app) 실행**: `AI PROMPTER.app`을 더블클릭합니다.
  - 처음 실행 시 서명되지 않은 앱이라 막히면, 앱을 **우클릭 → "열기"** 로 한 번만 허용해 주세요.
- **개발 모드 실행**(소스):
  ```bash
  cd backend
  .venv/bin/python3 -m app
  ```

실행하면 단일 창이 뜨고, **편집 화면**으로 시작합니다.

---

## 2. 화면 구성

| 화면 | 설명 |
|------|------|
| **편집 화면** | 왼쪽에 대본을 입력/수정, 오른쪽에 마이크·시작 위치·화면 설정 |
| **발표(프레젠테이션) 화면** | 대본을 큰 글씨로 표시하고, 말하는 대로 줄을 자동으로 넘김 |

오른쪽 위 **▶ 발표 시작** 으로 발표 화면에 들어가고, **← 편집**(또는 `Esc`)으로 돌아옵니다.

---

## 3. 대본 준비

- 왼쪽 **스크립트** 칸에 직접 입력하거나,
- 오른쪽 위 **📄 파일 가져오기** 로 문서를 불러옵니다.
  - 지원 형식: **`.txt` · `.md` · `.docx` · `.hwp`**
  - 구형 워드 `.doc`은 지원하지 않습니다 → 워드에서 `.docx`로 저장 후 가져오세요.

한 줄이 한 문장(또는 한 호흡) 단위가 되도록 줄바꿈하면 자동 넘김이 더 자연스럽습니다.

---

## 4. 마이크 설정

1. 오른쪽 **마이크** 드롭다운에서 사용할 입력 장치를 선택합니다. (목록이 비어 있으면 **새로고침**)
2. **입력 레벨** 미터를 보면서 시험 삼아 말해, 막대가 움직이는지 확인합니다.
   - 막대가 거의 안 움직이면: 시스템 입력 볼륨을 올리거나 다른 마이크를 선택하세요.

> macOS는 마이크 권한을 묻습니다 — **시스템 설정 → 개인정보 보호 및 보안 → 마이크**에서 앱을 허용해 주세요.

---

## 5. 발표 진행 (가장 중요)

1. **▶ 발표 시작** 을 누르면 발표 화면으로 전환되고, **음성 인식이 자동으로 켜집니다.**
2. **음성 인식을 켠 뒤 약 2~3초 기다렸다가 말하기 시작하세요.**
   - 인식을 켜는 순간 음성 인식 모델을 메모리에 올립니다(첫 사용 시 로딩).
   - 이 1~2초 동안의 말소리는 인식이 준비 중이라 놓칠 수 있습니다.
   - 화면 상단에 **🎙 인식 중** 표시가 뜨고 잠깐 뒤부터 말하면 안정적입니다.
3. 대본을 또박또박 읽으면, **현재 줄을 다 읽은 직후(약 0.7초 뒤)** 다음 줄로 자동으로 넘어갑니다.
   - 살짝 여유를 둔 타이밍이라, 줄 끝을 다 읽기 전에 미리 넘어가지 않습니다.

### 단축키 (발표 화면)

| 키 | 동작 |
|----|------|
| `Space` | 음성 인식 켜기/끄기 |
| `↓` | 다음 줄로 수동 이동 |
| `↑` | 이전 줄로 수동 이동 |
| `Esc` | 편집 화면으로 돌아가기 |

---

## 6. 화면·글꼴 꾸미기 (편집 화면 → 화면 설정)

발표 화면과 스크립트 화면 양쪽에 실시간으로 적용됩니다.

| 항목 | 설명 |
|------|------|
| **글자 크기** | 24~72px |
| **줄 간격** | 줄 사이 간격 |
| **텍스트 폭** | 본문이 차지하는 가로 폭 |
| **글꼴** | 기본(시스템) · Paperlogy · **설치된 폰트**(내 PC에 설치된 글꼴 자동 목록) |
| **굵기** | 가늘게 ~ 매우 굵게 |
| **글자 색** | 본문 글자 색 |
| **배경 색** | 발표 화면 배경 색 |

> 글자 색과 배경 색은 **대비가 잘 되도록** 함께 골라주세요(예: 어두운 배경 + 밝은 글자).

---

## 7. 음성 인식이 잘 되게 하는 요령

- **조용한 환경**에서, 마이크와 **너무 멀지 않게** 말합니다.
- 작은 소리도 인식되도록 입력 감도를 높여 두었지만, 그래도 **입력 레벨 미터가 반응하는 크기**로 말하는 게 좋습니다.
- 대본과 **같은 문장**을 읽을수록 줄 넘김이 정확합니다(즉흥 멘트는 매칭이 약함).
- 너무 빠르게 읽으면 줄을 건너뛸 수 있으니 자연스러운 속도로 읽으세요.
- 줄이 안 넘어갈 때 점검:
  1. 오른쪽 **자동 진행 켜짐** 상태인지 확인
  2. 올바른 **마이크**가 선택됐는지, **입력 레벨**이 반응하는지 확인
  3. 발표 화면 상단이 **🎙 인식 중** 인지 확인(아니면 `Space`로 켜기)

---

## 8. 인식 속도 vs 정확도 (모델 선택)

기본 모델은 **속도 우선(`openai/whisper-small`)** 입니다. 더 정확하게 쓰고 싶으면
`backend/.env` 의 한 줄만 바꾸세요:

```dotenv
# 빠름(기본)
TRANSFORMERS_WHISPER_MODEL_ID=openai/whisper-small
# 속도·정확도 균형
# TRANSFORMERS_WHISPER_MODEL_ID=openai/whisper-medium
# 한국어 최고 정확(느림)
# TRANSFORMERS_WHISPER_MODEL_ID=ghost613/whisper-large-v3-turbo-korean
```

- 개발 모드는 저장 후 다시 실행하면 적용됩니다.
- 패키지 앱(.app)에 반영하려면 재빌드가 필요합니다(`./build_macos.sh`).
- 모델은 첫 사용 시 자동으로 내려받아 캐시되며, 이후에는 빠르게 로드됩니다.

---

## 9. 자주 묻는 문제

| 증상 | 해결 |
|------|------|
| 마이크가 목록에 없음 | 시스템 마이크 권한 허용 → 앱에서 **새로고침** |
| 말해도 줄이 안 넘어감 | 자동 진행 켜짐 + 마이크/입력 레벨 + 🎙 인식 중 확인 |
| 인식이 자꾸 틀림 | 모델을 `whisper-medium` 또는 `ghost613`으로 변경(8번) |
| 줄이 너무 빨리/늦게 넘어감 | 자연스러운 속도로 읽기. 타이밍 조정이 필요하면 문의 |
| 글꼴이 안 바뀜 | "설치된 폰트" 목록에서 고르기(이모지·심볼 폰트는 제외됨) |
| 첫 인식이 느림 | 모델 로딩 때문 — 인식 켜고 2~3초 뒤부터 말하기 |
